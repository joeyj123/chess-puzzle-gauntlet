## Root causes fixed this session (for reference)

- **"Failed to set indexed property [0] on CSSStyleDeclaration"** — `customLightSquareStyle`
  and `customDarkSquareStyle` in LiveChess + MultiplayerDuel were passed raw hex strings
  (`boardTheme.light = '#f0d9b5'`). The main app correctly used `{ backgroundColor: ... }`.
  Fixed in LiveChess, MultiplayerDuel, ComputerChess, GameReview.

- **Moves not syncing in LiveChess** — Supabase JS v2 query builder is lazy: `.update().eq()`
  constructs the query but never sends it without `.then()` or `await`. All fire-and-forget
  updates now have `.then(() => {})`.

- **Turn/check indicator overlap** — Two separate DOM elements stacked when both were visible.
  Merged into one `.chess-status-bar` element that transitions between states.

## Known gotchas
- User runs commands in **PowerShell 5.1** — `&&` is NOT a valid statement
  separator. Use `;` or separate lines.
- The sandbox shell has no GitHub credentials — `git push` must be run by the
  user in their own terminal.
- **Don't run `git reset`/index-modifying commands from the sandbox bash** —
  previously caused a stale `.git/index.lock`.
- Sandbox bash file mounts can be **stale/truncated** for recently-edited files —
  always verify via `Read` before trusting a sandbox build error.
- `lichess_db_puzzle.csv` (1.1GB) — gitignored, not needed unless regenerating puzzles.
- `public/stockfish.js` — gitignored, auto-copied from `node_modules/stockfish` by
  the vite plugin on every build/dev start. Must run `npm install` before first dev/build.
